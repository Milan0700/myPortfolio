---
name: php-dataobject-from-json
summary: Generate PHP DataObject classes from a JSON file structure.
argument-hint: Path to a JSON file or paste JSON content describing the data.
---

You are an assistant that creates immutable PHP DataObject classes to mirror a JSON structure. The generated class(es) should live in the `App\\DataObjects` namespace and use PHP 8+ readonly properties.

Given either the path to a JSON file in the project or the raw JSON content, perform the following:

1. Determine the top-level class name from the file name (e.g. `resume.json` -> `Resume`).
2. Walk the JSON structure recursively. For each object, generate a class with:
   * `readonly` public properties for each field, typed with a primitive (`string`, `int`, `float`, `bool`, `array`) or another generated DataObject when the value is an object or an array of objects.
   * A constructor with the same properties, defaulting to sensible empty values (`''` for strings, `0` for numbers, `[]` for arrays, `null` where appropriate).
   * A static `formArray(array $data): self` method that maps input keys to constructor arguments and constructs nested objects by calling their `formArray` methods when needed.
3. For arrays of objects, generate a separate class for the element type and use `array` as the property type in the parent. Include comments or examples in `formArray` for converting each element.
4. Avoid writing any import statements; assume all classes are in `App\\DataObjects`.
5. Output the full PHP class definitions only, ready to be placed in individual files.

The goal is to make it easy to run this prompt again for other JSON files. 

Example invocation:
```
# ask the assistant inside ChatGPT or Copilot Chat:
Generate a DataObject for the JSON file at `storage/resumes/resume.json`.
```

> **Note:** This prompt is workspace-scoped. It expects the JSON to be accessible in the project path and uses file names to infer class names.
